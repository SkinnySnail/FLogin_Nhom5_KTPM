Kế hoạch Hành động
Đồ án Kiểm Thử (5 thành viên)
1. Khoa – Nhóm trưởng & CI/CD Lead (Câu 5)
1.1 Mục tiêu
Thiết lập toàn bộ hạ tầng kỹ thuật (repository, dependencies) cho việc kiểm thử.
Chịu trách nhiệm chính cho Câu 5: xây dựng pipeline CI/CD (GitHub Actions) chạy tự động toàn bộ test.
Tổng hợp báo cáo, screenshots, test reports từ các team thành file Bao_cao.pdf cuối cùng.
1.2 Phạm vi & ranh giới
Phạm vi: quản lý GitHub repo, file .github/workflows/ci.yml, cypress.config.js, cài đặt Page Object Model (POM), tổng hợp báo cáo.
Không làm: viết chi tiết các file *Test.java hoặc *.test.js (việc của team FE và BE).
1.3 Công việc chi tiết
A. Setup Repository
Tạo GitHub repo, setup .gitignore.
Đảm bảo pom.xml (Backend) và package.json (Frontend) chứa đủ thư viện testing (JUnit, Mockito, Jest, RTL, Cypress).
B. Test Case Design (Câu 1)
Chủ trì buổi họp phân tích Câu 1.
Tạo file TESTCASES.md hoặc Google Sheet chung để 2 team FE/BE điền test case chi tiết.
C. E2E & CI/CD Setup (Câu 5)
Cài đặt và cấu hình Cypress (cypress.config.js).
Viết file ProductPage.js (Page Object Model) theo mẫu đề bài.
Viết workflow CI/CD .github/workflows/ci.yml bao gồm:
actions/checkout@v2
actions/setup-java@v2 (Backend)
actions/setup-node@v2 (Frontend)
Chạy Backend Tests: mvnw clean test (Unit + Integration Test BE)
Chạy Frontend Tests: npm install && npm test (Unit + Integration Test FE)
Chạy E2E Tests: npm run test:e2e (Cypress, FE + BE)
(Nâng cao) Upload coverage reports bằng Codecov/JaCoCo.
Setup notification CI (Slack/GitHub) khi test fail.
Kiểm tra cross-browser cho E2E test (Chrome, Firefox) nếu cần.
D. Tổng hợp & Báo cáo
Review PRs của các team.
Thu thập screenshots, coverage report từ các team.
Viết file Bao_cao.pdf theo yêu cầu đề bài, bao gồm hình ảnh minh họa, test reports, coverage.
1.4 Deliverables
PRs: chore/ci-cd-setup, docs/pom-setup.
Artefacts: .github/workflows/ci.yml, cypress/pages/ProductPage.js, TESTCASES.md, Bao_cao.pdf.
1.5 Tiêu chí hoàn thành (DoD)
PR merge vào main phải kích hoạt CI pipeline tự động.
Pipeline báo cáo “Passed” nếu tất cả test thành công.
Bao_cao.pdf tổng hợp đủ 5 phần, có hình minh họa, test report, coverage.
1.6 Phối hợp & phụ thuộc
FE (Thịnh, Thái): cung cấp npm test và npm run test:e2e.
BE (Tài, Hoàng): cung cấp mvnw clean test.
________________________________________
2. Thịnh & Thái – Frontend Test Team (Câu 1,2,3,4,5)
2.1 Mục tiêu
Hoàn thành toàn bộ phần kiểm thử Frontend (React) cho Login & Product.
Viết Unit Test (TDD) cho validation, coverage >= 90%.
Viết Integration Test cho component Login, ProductForm.
Viết kịch bản E2E Test (Cypress).
2.2 Phạm vi & ranh giới
Phạm vi: file *.test.js, *.spec.js, làm việc với Jest, RTL, Cypress.
Không làm: code Java, file .yml CI/CD.
2.3 Công việc chi tiết
A. Test Case Design (Câu 1)
Viết 5 test cases chi tiết cho Login (theo template Bảng 1).
Viết 5 test cases chi tiết cho Product (theo template Bảng 2).
Bao gồm happy path, negative tests, boundary, edge cases.
B. Unit Testing & TDD (Câu 2)
validation.test.js: test validateUsername(), validatePassword() (rỗng, quá ngắn/dài, sai format, hợp lệ).
productValidation.test.js: test validateProduct() (name rỗng, price/quantity min/max, description > 500, category hợp lệ).
C. Integration Testing (Câu 3)
Login.integration.test.js: render(<Login />), fireEvent.change, fireEvent.click, waitFor error/success messages.
ProductForm.integration.test.js: tương tự, test form CRUD.
D. Mock Testing (Câu 4)
Login.mock.test.js: jest.mock('../services/authService') – test login success/fail.
Product.mock.test.js: mock productService – test Create/Get/Update/Delete Product.
E. E2E Automation (Câu 5)
login.e2e.spec.js: Cypress test flow login success/fail.
product.e2e.spec.js: Cypress test flow CRUD Product với POM ProductPage.js.
F. Bổ sung mới
Unit Test edge cases: ký tự đặc biệt, khoảng trắng đầu/cuối, max/min length.
Integration Test với mockService, dữ liệu rỗng/lớn/trùng tên.
E2E Test thêm login failure, product CRUD failure, search/filter.
Documentation & Evidence: FE_Test_Report.md, screenshots, video.
2.4 Deliverables
PRs: feat/fe-test-cases, feat/fe-unit-tests, feat/fe-integration-tests, feat/fe-e2e-scenarios.
Artefacts: file *.test.js, *.spec.js, screenshot coverage >= 90%, video E2E test.
2.5 Tiêu chí hoàn thành (DoD)
npm test và npm run test:e2e chạy pass trên CI.
Toàn bộ test cases (Câu 1) được implement.
Coverage đạt yêu cầu.
2.6 Phối hợp & phụ thuộc
Khoa (Lead): setup CI pipeline & POM E2E.
BE (Tài, Hoàng): cung cấp DTOs / JSON structure để mock chính xác.
________________________________________
3. Tài & Hoàng – Backend Test Team (Câu 1,2,3,4)
3.1 Mục tiêu
Hoàn thành kiểm thử Backend (Spring Boot) cho Login & Product.
Viết Unit Test tầng Service (coverage >= 85%).
Viết Integration Test tầng Controller (API endpoints).
3.2 Phạm vi & ranh giới
Phạm vi: file *Test.java làm việc với JUnit 5, Mockito, MockMvc.
Không làm: React code, file .yml CI/CD.
3.3 Công việc chi tiết
A. Test Case Design (Câu 1)
Viết test case logic backend: username không tồn tại, password sai, validation server.
B. Unit Testing & TDD (Câu 2)
AuthServiceTest.java: @Mock UserRepository, @InjectMocks AuthService. Test authenticate(): success, user không tồn tại, sai password.
ProductServiceTest.java: @Mock ProductRepository, @InjectMocks ProductService. Test CRUD: create, getById, update, delete. Verify save gọi 1 lần.
C. Integration Testing (Câu 3)
AuthControllerIntegrationTest.java: @WebMvcTest(AuthController.class), @MockBean AuthService. MockMvc POST /api/auth/login.
ProductControllerIntegrationTest.java: GET / POST / PUT / DELETE endpoints, MockMvc, check status, response structure.
D. Mock Testing (Câu 4)
Backend mock củng cố từ Câu 2,3.
ProductServiceMockTest.java: @Mock ProductRepository.
AuthControllerMockTest.java: @MockBean AuthService.
E. Bổ sung mới
Unit Test edge cases backend: boundary values, invalid input.
Integration Test tất cả API endpoints (GET, POST, PUT, DELETE, login) với nhiều scenario.
Mock repository/controller layer, verify call đúng.
JaCoCo report >= 85%, lưu log chi tiết.
3.4 Deliverables
PRs: feat/be-unit-tests, feat/be-integration-tests.
Artefacts: file *Test.java, screenshot coverage >= 85%.
3.5 Tiêu chí hoàn thành (DoD)
mvnw clean test pass trong CI pipeline.
Coverage đạt yêu cầu.
API endpoints ready cho team FE và E2E test.
3.6 Phối hợp & phụ thuộc
Khoa: setup CI pipeline để chạy mvnw clean test.
FE (Thịnh/Thái): cung cấp DTOs / JSON structure.
________________________________________
4. Tất cả team – Phối hợp & Giao tiếp
Dùng Google Sheet / TESTCASES.md để track test case status.
Merge PR chỉ khi test pass local + CI.
Ghi chú tất cả lỗi trong bug tracker (GitHub Issues).
Mỗi team cung cấp evidence (screenshots, coverage, video) cho Khoa tổng hợp.
________________________________________

Code & Test: tất cả *.java, *.test.js, *.spec.js, ProductPage.js, CI/CD workflow.
Reports: FE_Test_Report.md, BE_Test_Report.md, Bao_cao.pdf.
Evidence: screenshots, video E2E test.
CI/CD: workflow hoạt động tự động, báo cáo test & coverage.

feature/be-login, feature/be-product
